WORKING WITH NAVMESHES

A Navigational Mesh is a data structure which describes the walkable surfaces of the game world and allows finding a path from one walkable location to another.

  • Enable the Navigation window: Window > Navigation
  • NavMesh baking: The process of creating a nav mesh in Unity.
  • Baking & Clearing: NavMeshes can be generated & cleared with these buttons. The NavMeshes are stored in Assets/Name_Of_Scene.
  • Navigation Static: All gameobjects must be set to Navigation Static to be included in NavMesh baking.
  • Carving: Navigation Static gameobjects punch holes in the mesh through a process known as carving.
  • A nav mesh is an approximation of the walkable surface. Sometimes you have to leave a little extra room.
  • Min Region: NavMesh regions with surface area smaller than this number will be culled away.
  • Height Map: Enabling the Bake setting, Height Map, can improve vertical fidelity (at cost of performance).

NAVMESH AGENT COMPONENT

The NavMesh Agent component handles both the pathfinding and the movement control of a character.

  • NavMesh Agents will create paths to goals within the Nav Mesh.
  • Avoidance: NavMesh Agents will avoid obstacles & other agents.
  • A*: Unity's NavMesh Agent component uses the A* algorithm to pathfind.
  • Positioning: NavMesh Agents should be positioned at their base. If not, adjust the offset property to compensate.

OFF-MESH LINKS

  • Links can either be generated or manually created
  • Generate: Select Generate Off-mesh Links in Navigation > Object for each gameobject.
  • Bake: Set Drop Height & Jump Distance before baking
  • Drop Height & Jump Distance: These distances should be exaggerated slightly to account for small deviations created in baking process.
  • Manual creation: Create start & end point gameobjects & attach the OffMesh Link component to one.
  • Placing start & end points: You want solid black rings & line. If it is faded, or a ring is missing, adjust access point closer to the surface.

NAVMESH OBSTACLE COMPONENT

The NavMesh Obstacle component allows you to describe moving obstacles that the agents should avoid while navigating the world.

  • Add this component to any non-static gameobject that you want the NavMesh Agent to avoid.
  • Carve only stationary: While moving, agents will try to avoid, but once it is stationary, it will carve a hole in the Nav Mesh, forcing an agent to find a different path.
  • Carve when moved: If unselected, a gameobject will continuously carve the Nav Mesh. Only ideal for large, slow moving obstacles.
  • Time to stationary: Sets the delay used before re-carving Nav Mesh.
