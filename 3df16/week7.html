LIGHTS (GLOBAL ILLUMINATION)

  • REMEMBER: Non-moving geometry (avoid setting very large, or very complex gameobjects to static!) and lighting should be set to Static in the top-right corner of the Inspector.
  • If you are working on a modest laptop, you may want to deselect auto baking at the bottom of the Lighting pane. Be sure to use the Bake button when you want to test your lights.

NORMAL MAPS

  • Unity supports bump mapping through the use of normal maps
  • In addition a greyscale image can be used as a height map to provide extra dimension (More expensive performance)
  • Like textures, normal maps can be exported through your 3D modeling software or created by standalone tools (Photoshop, xNormal)
  • Unlike textures, they need to be converted when imported by changing the type from 'Texture' to 'Normal Map' and re-importing
  • It's possible to use mismatching Albedo textures and normal maps

LIGHT COOKIES

  • Cookies can make your lights look better
  • Use a grayscale image as a texture (lighter areas emit light) for spotlights and direction lights. Create a cubemap for point lights.
  • Cookies can only be used on non-static lights
  • IES asset (free download of 4 light cookies)

EMISSIVE MATERIALS

  • Can be used to emit lights
  • Grayscale textures can be used as emission
  • Scale the emission value above 1 for HDR (glowy results)
  • GameObjects must be static for their emitted light to affect other surfaces

VOLUMETRIC LIGHTING

  • Fantastic scripts you can use to create volumetric lighting with spotlights can be found here (link)

IMAGE EFFECTS
